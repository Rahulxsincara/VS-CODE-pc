Educational Resource Library Database Structure

This Entity-Relationship model illustrates the database structure for the Educational Resource Library system. It shows the main entities and their relationships.

RESOURCE
resource_id INT PK
title VARCHAR(255)
description TEXT
author VARCHAR(100)
file_url VARCHAR(500)
file_size INT
download_count INT
category_id INT FK
uploaded_by INT FK
created_at TIMESTAMP
USER
user_id INT PK
username VARCHAR(50)
email VARCHAR(100)
password_hash VARCHAR(255)
role ENUM('admin','user')
created_at TIMESTAMP
CATEGORY
category_id INT PK
name VARCHAR(100)
description TEXT
DOWNLOAD
download_id INT PK
user_id INT FK
resource_id INT FK
download_date TIMESTAMP

Relationships

USER - RESOURCE (1:N): One user can upload many resources, but each resource is uploaded by one user.

CATEGORY - RESOURCE (1:N): One category can contain many resources, but each resource belongs to one category.

USER - DOWNLOAD (1:N): One user can download many resources, creating many download records.

RESOURCE - DOWNLOAD (1:N): One resource can be downloaded many times, creating many download records.
